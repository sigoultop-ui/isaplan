|---|---|---|
| `index.html` | 랜딩 페이지 | 완료 |

### 현재 기능

- ISA 유형 선택 (일반형 / 서민·농어민형)
- 월 납입액, 기대 수익률, 연 납입 증액, 시작 연도 설정

- 누적 원금·수익·총자산 요약 카드

- 단일 시나리오만 비교 가능 (낙관/중립/비관 없음)

- 결과 공유·내보내기 없음

- 투자 기간이 10년으로 고정

- SEO, 메타 태그 없음
- 접근성(ARIA) 미적용

## 2. 개선 목표

```
사용자가 자신의 투자 조건을 입력하고,
```

## Phase 1 — 계산기 기능 고도화

> 난이도: ★★★☆☆ | 우선순위: 🔴 높음

### 1-1. 투자 기간 동적 설정

- 현재 10년 고정 → **1년 ~ 30년** 슬라이더로 변경
- 의무 가입 3년 미만 시 경고 표시

```js
// 변경 포인트: calculator.html
const YEARS = parseInt($('years').value); // 기존 고정값 10 제거
```

### 1-2. 다중 시나리오 비교

- **낙관 / 중립 / 비관** 3개 수익률 동시 계산
- 차트에 3개 라인 오버레이
- 테이블 탭 전환으로 시나리오별 상세 내역 확인

| 시나리오 | 기본 수익률 |
|---|---|
| 낙관 | 사용자 입력 + 3% |
| 중립 | 사용자 입력 |
| 비관 | 사용자 입력 - 3% |

### 1-3. ETF 비용 반영

- 연간 운용 보수 입력 필드 추가 (기본값 0.05%)
- 매매 수수료 반영 옵션 (기본값 0.015%)
- 실질 수익률 = 기대 수익률 - 운용 보수

### 1-4. 인플레이션 조정 실질 수익

- 인플레이션율 입력 (기본값 2.5%)
- 명목 수익 / 실질 수익 구분 표시
- "현재 화폐 가치 기준" 자산 표시

### 1-5. 납입 이월 한도 계산

- ISA 미납분 이월 계산 로직 추가
- 연 2,000만원 미납 시 다음 연도 추가 납입 가능액 표시
- 최대 5년 이월, 총 1억원 한도 반영

### 1-6. 목표 금액 역산 기능

- "목표 자산" 입력 시 → 필요 월 납입액 자동 계산
- 현재 납입액으로 목표 달성 가능 기간 표시

## Phase 2 — UX·UI 품질 개선

> 난이도: ★★☆☆☆ | 우선순위: 🟠 중간

### 2-1. 모바일 반응형 최적화

- 계산기 컨트롤 패널 모바일 전용 레이아웃 (아코디언 섹션)
- 테이블 가로 스크롤 → 모바일에서 주요 컬럼만 표시 + 상세보기 토글
- 고정 하단 요약바 (총자산 / 수익률 항상 표시)

### 2-2. 차트 고도화

- 차트 유형 전환 버튼 (바 차트 / 영역 차트 / 파이 차트)
- 차트 클릭 시 해당 연도 상세 패널 팝업
- 애니메이션 진입 효과 개선

### 2-3. 입력 UX 개선

- 숫자 직접 입력 + 슬라이더 양방향 동기화 (현재는 슬라이더만)
- 프리셋 버튼: "사회초년생 (30만/월)", "직장인 평균 (70만/월)", "고소득자 (150만/월)"
- 입력값 유효성 검사 및 에러 메시지

### 2-4. 접근성 (Accessibility)

- `aria-label`, `role` 속성 추가
- 키보드 네비게이션 지원
- 고대비 모드 지원
- 스크린 리더 친화적 테이블 구조

### 2-5. 다크/라이트 모드 전환

- 시스템 설정 자동 감지 (`prefers-color-scheme`)
- 수동 토글 버튼 (헤더 우측)
- `localStorage`로 설정 유지

### 2-6. 로딩 성능 최적화

- Chart.js CDN → 로컬 번들로 전환 (오프라인 지원)
- 폰트 `font-display: swap` 적용
- 중요 CSS 인라인화, 나머지 비동기 로드

## Phase 3 — 데이터 저장 및 공유

> 난이도: ★★★★☆ | 우선순위: 🟡 중간

### 3-1. localStorage 저장

- 마지막 입력값 자동 저장 (새로고침 후 복원)
- 여러 계획안 저장·불러오기 (최대 5개)
- 저장된 계획안 목록 사이드패널

```js
// 저장 예시
const plan = { monthly, rate, increment, isaType, startYear };
localStorage.setItem('isa_plan_1', JSON.stringify(plan));
```

### 3-2. URL 공유 기능

- 입력값을 URL 쿼리스트링으로 인코딩
- "공유 링크 복사" 버튼 → 클립보드에 복사
- 링크 접속 시 해당 설정 자동 복원

```
예시: calculator.html?monthly=70&rate=8&years=10&isa=general
```

### 3-3. 결과 내보내기

- **PNG 이미지** 저장 (차트 + 요약 카드)
- **CSV 다운로드** (연도별 테이블 데이터)
- **PDF 출력** 최적화 (`@media print` 스타일)

### 3-4. 카카오톡 / 트위터 공유

- 오픈 그래프(OG) 메타 태그 동적 생성
- 소셜 공유 버튼 (카카오, 트위터/X, 링크 복사)

## Phase 4 — 프로젝트 구조 현대화

> 난이도: ★★★★★ | 우선순위: 🟢 낮음 (장기)

### 4-1. 빌드 도구 도입

현재 단일 HTML 파일 → **Vite** 기반 프로젝트로 전환

```
권장 스택
- Vite (번들러)
- Vanilla JS → TypeScript 전환
- CSS Modules 또는 Tailwind CSS
```

### 4-2. 모듈 분리

```
src/
├── main.ts              # 진입점
├── calculator/
│   ├── engine.ts        # 순수 계산 로직 (UI 분리)
│   ├── chart.ts         # Chart.js 래퍼
│   └── table.ts         # 테이블 렌더링
├── store/
│   └── settings.ts      # localStorage 상태 관리
├── utils/
│   ├── format.ts        # 숫자 포맷 헬퍼
│   └── url.ts           # 쿼리스트링 파싱
└── pages/
    ├── landing/
    └── calculator/
```

### 4-3. 테스트 추가

- **단위 테스트**: 계산 엔진 로직 (Vitest)
- **E2E 테스트**: 슬라이더 조작 → 결과 검증 (Playwright)

```ts
// 테스트 예시
test('10년 복리 계산 정확도', () => {
  const result = calculate({ monthly: 50, rate: 0.08, years: 10 });
  expect(result.cumAsset).toBeCloseTo(91567, 0); // 만원
});
```

### 4-4. CI/CD 파이프라인

- GitHub Actions로 자동 빌드·배포
- PR 시 테스트 자동 실행
- GitHub Pages 또는 Vercel 배포
